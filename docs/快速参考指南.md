# MyBatis 源码学习环境快速参考指南

## 项目结构

```
mybatis-learn/
├── mybatis-3/              # MyBatis 源码
├── mybatis-main-dir/       # 测试项目
├── parent/                 # 父项目
├── spring/                 # Spring 集成
├── docs/                   # 文档
├── pom.xml                 # 根 pom.xml
└── build-and-test.sh       # 构建脚本
```

## 快速开始

### 1. 编译 MyBatis 源码

```bash
cd mybatis-3
mvn clean install -DskipTests
```

### 2. 运行测试

```bash
cd mybatis-main-dir
mvn test
```

### 3. 使用自动化脚本

```bash
./build-and-test.sh          # 完整构建和测试
./build-and-test.sh test     # 只运行测试
./build-and-test.sh source   # 源码依赖模式
```

## 关键配置

### pom.xml 配置

```xml
<dependency>
    <groupId>org.mybatis</groupId>
    <artifactId>mybatis</artifactId>
    <version>3.5.4-my-debug-snapshot</version>
    <scope>system</scope>
    <systemPath>${project.basedir}/../mybatis-3/target/mybatis-3.5.4-my-debug-snapshot.jar</systemPath>
</dependency>

<!-- 传递依赖 -->
<dependency>
    <groupId>ognl</groupId>
    <artifactId>ognl</artifactId>
    <version>3.2.12</version>
</dependency>
<dependency>
    <groupId>org.javassist</groupId>
    <artifactId>javassist</artifactId>
    <version>3.26.0-GA</version>
</dependency>
```

### MyBatis 配置

```xml
<!-- mybatis-config.xml -->
<mappers>
    <mapper class="com.zero.app.dao.UserDao"/>
</mappers>
```

## 验证方法

### 检查依赖来源

```bash
mvn dependency:tree
```

**正确输出**：
```
org.mybatis:mybatis:jar:3.5.4-my-debug-snapshot:system
```

**错误输出**：
```
org.mybatis:mybatis:jar:3.5.4-my-debug-snapshot:compile
```

### 检查 JAR 文件

```bash
ls -la mybatis-3/target/mybatis-3.5.4-my-debug-snapshot.jar
```

## 常见问题

### 问题 1：`java: 程序包ognl不存在`

**解决**：添加 OGNL 依赖

```xml
<dependency>
    <groupId>ognl</groupId>
    <artifactId>ognl</artifactId>
    <version>3.2.12</version>
</dependency>
```

### 问题 2：`Type interface com.zero.app.dao.UserDao is not known to the MapperRegistry`

**解决**：检查 MyBatis 配置文件中的 Mapper 类名

```xml
<mappers>
    <mapper class="com.zero.app.dao.UserDao"/>  <!-- 确保类名正确 -->
</mappers>
```

### 问题 3：`Could not find artifact org.mybatis:mybatis:jar:3.5.4-my-debug-snapshot`

**解决**：先编译 MyBatis 源码

```bash
cd mybatis-3
mvn clean install -DskipTests
```

## 开发工作流

### 修改 MyBatis 源码

1. 修改 `mybatis-3/src/main/java/` 中的源码
2. 重新编译：`cd mybatis-3 && mvn clean install -DskipTests`
3. 运行测试：`cd mybatis-main-dir && mvn test`

### 调试源码

1. 在 IDE 中设置断点
2. 运行测试程序
3. 观察 MyBatis 内部执行流程

### 添加新功能

1. 在 `mybatis-3` 中添加新功能
2. 在 `mybatis-main-dir` 中编写测试用例
3. 验证功能正确性

## 有用的命令

```bash
# 清理和编译
mvn clean compile

# 运行测试
mvn test

# 检查依赖
mvn dependency:tree

# 分析依赖
mvn dependency:analyze

# 运行主程序
mvn exec:java -Dexec.mainClass="com.zero.app.App"
```

## 注意事项

1. **每次修改源码后都要重新编译**
2. **确保 JAR 文件路径正确**
3. **验证依赖来源**
4. **保持配置文件同步**
